At the start of this semester I thought using AI in programming was mostly cheating or a shortcut that prevented real learning. That perspective changed as I started using it in more specific and intentional ways. One of the most helpful uses was having AI clean up my code and add proper documentation like javadoc headers and inline comments. This did not write the logic for me but it made my code more readable and professional without taking away from the actual problem solving. I also used it when researching topics I had not seen before, like looking up how a design pattern worked before trying to implement it myself.
Going forward I plan to keep using AI in those same specific ways. When I start working with a new framework or language I will use it to get up to speed faster instead of spending hours reading documentation alone. I will also continue using it to improve code readability, adding headers, cleaning up formatting, and making sure my code follows good conventions. This class showed me that AI is most valuable as a support tool when you already understand what you are building. The goal is to use it to work smarter, not to replace the thinking and problem solving that actually builds your skills.
